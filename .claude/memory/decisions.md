## ADR-001: Next.js 15 App Router 선택

**날짜**: 2026-01-25

**상태**: 승인됨

**컨텍스트**:
- 라이브 코딩 콘텐츠로 사용될 쇼핑몰 스캘레톤
- 비개발자가 AI로 커스터마이징 가능해야 함

**결정**:
Next.js 15 App Router를 선택

**근거**:
1. React 19 기능 활용 가능
2. 스트리밍 SSR 지원
3. 서버 컴포넌트로 성능 최적화
4. Vercel과 최적 통합

**대안 검토**:
- Remix: 데이터 로딩 패턴 좋지만 생태계 작음
- SvelteKit: 학습 곡선 있음

## ADR-002: Supabase 선택

**날짜**: 2026-01-25

**상태**: 승인됨

**컨텍스트**:
- BaaS로 빠른 개발 필요
- Auth + DB + Storage 통합 필요
- RLS 기반 보안 필요

**결정**:
Supabase 선택 (PostgreSQL + Auth + Storage)

**근거**:
1. Auth + DB + Storage 원스톱 솔루션
2. RLS로 데이터 보안 보장
3. Vercel과 쉬운 연동
4. 오픈소스로 락인 리스크 낮음

**리스크**:
- RLS 정책 설정 복잡도 (Top 리스크로 관리)

## ADR-004: shadcn/ui 선택

**날짜**: 2026-01-25

**상태**: 승인됨

**컨텍스트**:
- 빠른 UI 개발 필요
- AI 커스터마이징 용이해야 함

**결정**:
shadcn/ui 선택 (복사-붙여넣기 방식)

**근거**:
1. 컴포넌트 코드가 프로젝트에 직접 포함
2. 완전한 커스터마이징 가능
3. Tailwind CSS 기반
4. 접근성 기본 지원

**대안 검토**:
- Material UI: 번들 크기 큼, 커스터마이징 어려움
- Chakra UI: 좋지만 shadcn/ui가 더 가벼움

## ADR-005: Zustand 상태관리 선택

**날짜**: 2026-01-25

**상태**: 승인됨

**컨텍스트**:
- 클라이언트 상태관리 필요 (장바구니 등)
- 비개발자도 이해할 수 있어야 함

**결정**:
Zustand 선택

**근거**:
1. API가 매우 간단
2. 보일러플레이트 최소
3. 타입스크립트 지원 우수
4. persist 미들웨어로 로컬 스토리지 연동 쉬움

**대안 검토**:
- Redux Toolkit: 기능 많지만 복잡
- Jotai: 좋지만 Zustand가 더 직관적
