# P8-S9-T1: 수동 테스트 체크리스트

## 준비

1. 개발 서버 시작:
   ```bash
   cd worktree/phase-8-production
   npm run dev
   ```

2. 로그인 후 에디터 페이지 접속:
   - URL: `http://localhost:3000/editor/[guidebook-id]`
   - 기존 가이드북 ID 또는 새로 생성

## 테스트 시나리오

### 1. 디바이스 프레임 렌더링
- [ ] 오른쪽 Preview 패널에 iPhone 프레임이 표시됨
- [ ] 노치가 상단 중앙에 표시됨
- [ ] 상태바(시간, WiFi, 배터리)가 표시됨
- [ ] 홈 인디케이터가 하단 중앙에 표시됨
- [ ] 프레임이 부드러운 곡선(rounded)을 가짐

### 2. 디바이스 크기 선택
- [ ] "iPhone SE" 버튼 클릭 시 프레임 크기가 375px로 변경됨
- [ ] "iPhone 14" 버튼 클릭 시 프레임 크기가 390px로 변경됨 (노치 표시)
- [ ] "iPad" 버튼 클릭 시 프레임 크기가 768px로 변경됨 (노치 없음)
- [ ] 선택된 버튼에 primary 색상이 적용됨

### 3. 라이트/다크 모드 토글
- [ ] 다크 모드 버튼 클릭 시 배경이 어두워짐
- [ ] 텍스트 색상이 흰색으로 변경됨
- [ ] 라이트 모드 버튼 클릭 시 원래대로 돌아옴

### 4. 블록 렌더링
- [ ] 추가한 블록이 프레임 내부에 표시됨
- [ ] Hero 블록이 이미지와 텍스트를 정확히 표시함
- [ ] QuickInfo 블록이 체크인/아웃 정보를 표시함
- [ ] 기타 블록들이 게스트 뷰어와 동일하게 렌더링됨

### 5. 실시간 동기화
- [ ] 왼쪽 에디터에서 Hero 제목 변경 시 프레임에 즉시 반영됨
- [ ] 이미지 업로드 시 프레임에 즉시 표시됨
- [ ] 블록 순서 변경 시 프레임에서도 순서가 변경됨

### 6. 블록 선택
- [ ] TOC에서 블록 클릭 시 프레임에서 해당 블록이 하이라이트됨
- [ ] 프레임에서 블록 클릭 시 왼쪽 에디터에서 해당 블록이 선택됨
- [ ] 하이라이트가 ring + outline으로 명확히 표시됨

### 7. 빈 상태
- [ ] 블록이 없을 때 "블록을 추가하여 시작하세요" 메시지가 표시됨
- [ ] 빈 상태 아이콘이 표시됨

### 8. 접근성
- [ ] Tab 키로 디바이스 버튼 간 이동 가능
- [ ] Enter 키로 디바이스 선택 가능
- [ ] Tab 키로 프레임 내 블록 간 이동 가능
- [ ] Enter/Space 키로 블록 선택 가능

### 9. 반응형 (모바일)
- [ ] 모바일(<lg)에서 편집/미리보기 탭 전환이 작동함
- [ ] 탭 전환 시 Preview가 전체 화면으로 표시됨
- [ ] 디바이스 버튼 아이콘만 표시됨 (텍스트 숨김)

### 10. 성능
- [ ] 블록 변경 시 렌더링 지연이 없음
- [ ] 디바이스 크기 변경이 부드럽게 애니메이션됨
- [ ] 다크 모드 전환이 즉시 적용됨
