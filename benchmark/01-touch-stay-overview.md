## 1. 서비스 개요

### 기본 정보

| 항목 | 내용 |
|------|------|
| **서비스명** | Touch Stay |
| **웹사이트** | [touchstay.com](https://touchstay.com) |
| **설립** | 영국 기반 SaaS 스타트업 |
| **인수** | 2024년 5월 14일 Itrinity에 인수됨 |
| **주요 시장** | 미국, 유럽, 호주 (영어권 중심) |

### 한 줄 정의

> **"디지털 게스트 가이드북 플랫폼"** - 숙박 시설의 정보를 모바일 친화적인 웹 앱 형태로 제공하여 게스트 경험을 개선하고 호스트의 반복 문의를 줄여주는 서비스

### 핵심 가치 제안

1. **게스트 경험 향상**: 체크인 전부터 퇴실까지 필요한 모든 정보를 한 곳에
2. **호스트 시간 절약**: 92%의 사용자가 시간 절약 효과 체감
3. **친환경**: 종이 매뉴얼 대체로 친환경적
4. **24/7 접근성**: 오프라인에서도 작동, 앱 다운로드 불필요

### 사용자 만족도

- **92%** 사용자가 시간 절약 및 고객 만족도 향상 보고
- Trustpilot에서 높은 평점 유지

## 4. 템플릿 구조

### 4.1 가이드북 계층 구조 (3-Level System)

```
📁 Category (카테고리) - 최상위 메뉴
   └── 📂 Subcategory (서브카테고리) - 소제목
         └── 📄 Topic (토픽) - 실제 콘텐츠
```

**예시:**
```
📁 Your Local Area Guide (카테고리)
   └── 📂 Restaurants (서브카테고리)
         ├── 📄 Best Italian Restaurant
         ├── 📄 Local Korean BBQ
         └── 📄 Breakfast Spots
   └── 📂 Things to Do (서브카테고리)
         ├── 📄 Beach Access
         └── 📄 Hiking Trails
```

## 6. 장단점 분석

### 6.1 강점 (Strengths)

| 강점 | 상세 |
|------|------|
| **AI 기능** | 60초 만에 Airbnb 링크로 가이드북 자동 생성 |
| **사용 편의성** | 코딩 필요 없음, 드래그 앤 드롭 |
| **오프라인 지원** | 앱 다운로드 없이 오프라인 접근 가능 |
| **다양한 템플릿** | 16개+ 산업별 템플릿 |
| **다국어 지원** | 자동 번역 기능 |
| **빠른 고객지원** | 사용자 리뷰에서 호평 |
| **지속적 업데이트** | 새 기능 정기 추가 |

### 6.2 약점 (Weaknesses) - ⭐ 차별화 기회

| 약점 | 상세 | 우리 서비스 기회 |
|------|------|-----------------|
| **PMS 연동 제한** | OwnerRez, BookingSync 위주 | 야놀자/여기어때 연동으로 차별화 |
| **스마트락 미지원** | 체크인/스마트락 접근 불가 | 도어락 연동 고려 |
| **개별 게스트 추적 불가** | 섹션별 통계만 제공 | 개별 행동 분석 제공 |
| **비디오 직접 업로드 불가** | YouTube 링크만 가능 | 직접 업로드 지원 |
| **WhatsApp 미지원** | SMS만 지원 | 카카오톡 연동 필수 |
| **URL 커스텀 제한** | 게스트 이름 등 포함 불가 | 개인화 URL 제공 |
| **뒤로가기 버튼 없음** | 지도 섹션에서 이전 화면 복귀 어려움 | UX 개선 |
| **한국어 UI 없음** | 영어 중심 | 완전한 한국어 서비스 |
| **한국 결제 미지원** | 해외 결제만 | 국내 카드/카카오페이 |

### 6.3 사용자 불만 사항 (피드백 포털 기반)

1. "게스트 이름이 URL에 포함되면 기억하기 쉬울 텐데..."
